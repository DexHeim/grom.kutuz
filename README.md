# ОСН «Гром» — сайт для GitHub Pages

Статичный сайт для RMRP, сервер №5 «Кутузовский»: FAQ, внутренние правила, ссылки, памятки, система повышения, калькулятор баллов, публичный состав и админка состава.

## Что изменено в v3

- Добавлена админка с входом по логину и паролю.
- Пароль хранится не открытым текстом, а PBKDF2-SHA256-хешем в `assets/js/admin-auth-config.js`.
- Добавлен генератор хеша: `tools/hash-password.html`.
- В админке оставлен ручной экспорт `roster.json` и добавлена опциональная запись в GitHub через GitHub API.
- Калькулятор переделан визуально: быстрые кнопки `+ / −`, поиск активностей, нормальная сводка, список внесённых активностей.
- Состав переделан визуально: сводка, фильтр по группе, фильтр по статусу, карточки сотрудников.

## Важное про безопасность

GitHub Pages — статический хостинг. У сайта нет своего сервера, базы данных и серверной проверки сессии.

Поэтому логин/пароль в `admin.html` защищает интерфейс от случайных людей, но не является полноценной серверной авторизацией. Реальная защита изменения сайта — доступ к репозиторию GitHub или GitHub token с правом записи.

Нельзя вставлять GitHub token в HTML, JS, README или Discord. Токен вводится только руками в админке и может храниться только в `sessionStorage` до закрытия вкладки.

## Как залить на GitHub Pages

1. Создай репозиторий на GitHub.
2. Загрузи все файлы из архива в корень репозитория.
3. Открой `Settings` → `Pages`.
4. Выбери `Deploy from a branch`.
5. Branch: `main`, Folder: `/root`.
6. Сохрани.

## Как сменить пароль админки

1. Открой `tools/hash-password.html`.
2. Введи логин, отображаемое имя и новый пароль.
3. Нажми `Сгенерировать`.
4. Скопируй получившийся объект.
5. Открой `assets/js/admin-auth-config.js`.
6. Замени объект внутри массива `users`.

Пример структуры:

```js
window.GROM_ADMIN_AUTH = {
  enabled: true,
  sessionMinutes: 120,
  users: [
    {
      login: 'admin',
      displayName: 'Администратор ОСН',
      algorithm: 'PBKDF2-SHA256',
      iterations: 210000,
      salt: '...',
      hash: '...'
    }
  ]
};
```

Чтобы добавить несколько админов, добавь несколько объектов в `users`.

## Как редактировать состав

### Вариант 1. Через админку и скачивание JSON

1. Открой `admin.html`.
2. Войди по логину и паролю.
3. Добавь, измени или удали сотрудников.
4. Нажми `Скачать`.
5. Замени скачанным файлом `data/roster.json` в репозитории.
6. Сделай commit.

### Вариант 2. Через админку и GitHub API

1. Создай GitHub token с доступом на запись содержимого репозитория.
2. Открой `admin.html`.
3. Заполни `owner`, `repo`, `branch`, `path`.
4. Вставь token.
5. Нажми `Запушить roster.json`.
6. Подожди деплой GitHub Pages.

Для `path` по умолчанию используется:

```text
data/roster.json
```

Для `branch` по умолчанию используется:

```text
main
```

### Вариант 3. Через JSON напрямую

Открой файл:

```text
data/roster.json
```

Структура:

```json
{
  "updated": "2026-06-09",
  "groups": [
    {
      "title": "Командование",
      "members": [
        {
          "name": "ФИО / ник",
          "rank": "звание",
          "position": "должность",
          "callsign": "позывной",
          "status": "active",
          "note": "примечание"
        }
      ]
    }
  ]
}
```

Статусы:

- `active` — активен;
- `trainee` — стажёр;
- `reserve` — резерв;
- `leave` — отпуск.

## Как менять баллы и систему повышения

Открой файл:

```text
assets/js/site-data.js
```

Там два массива:

- `GROM_ACTIVITIES` — активности калькулятора;
- `GROM_PROMOTION_STEPS` — ступени повышения.

Пример активности:

```js
{ id: 'fine', title: 'Штраф', points: 10, proof: 'Скриншот выписки штрафа' }
```

Меняешь `points`, `title` или `proof` — калькулятор сам подхватит новые данные.

## Как дать доступ другому человеку

Нормальный вариант — добавить человека в collaborators репозитория. Тогда он сможет редактировать файлы и коммитить изменения.

Если человек должен редактировать только состав через админку, ему всё равно нужен GitHub token или доступ к репозиторию. Один только пароль от `admin.html` не даёт права записи в GitHub.

## Как смотреть локально

Из-за загрузки `data/roster.json` через `fetch` лучше открывать сайт через локальный сервер:

```bash
python -m http.server 8000
```

Потом открыть:

```text
http://localhost:8000
```
