# ОСН «Гром» — сайт для GitHub Pages

Статичный сайт для RMRP, сервер №5 «Кутузовский»: FAQ, внутренние правила, ссылки, памятки, система повышения, калькулятор баллов и публичный состав.

## Что изменено во второй версии

- Сайт разбит на отдельные страницы, а не одну длинную простыню.
- Добавлена страница `roster.html` для публикации состава.
- Добавлена страница `admin.html` — локальный редактор состава с экспортом `data/roster.json`.
- Калькулятор переделан из широкой таблицы в карточки активностей, поэтому внутри блока нет горизонтальной прокрутки.
- Данные повышения и баллов вынесены в `assets/js/site-data.js`.

## Как залить на GitHub Pages

1. Создай репозиторий на GitHub.
2. Загрузи все файлы из архива в корень репозитория.
3. Открой `Settings` → `Pages`.
4. Выбери `Deploy from a branch`.
5. Branch: `main`, Folder: `/root`.
6. Сохрани. Через короткое время сайт появится по ссылке GitHub Pages.

## Как менять баллы и систему повышения

Открой файл:

```text
assets/js/site-data.js
```

Там два массива:

- `GROM_ACTIVITIES` — активности калькулятора;
- `GROM_PROMOTION_STEPS` — ступени повышения.

Пример активности:

```js
{ id: 'fine', title: 'Штраф', points: 10, proof: 'Скриншот выписки штрафа' }
```

Меняешь `points`, `title` или `proof` — калькулятор сам подхватит новые данные.

## Как редактировать состав

### Вариант 1. Через JSON напрямую

Открой файл:

```text
data/roster.json
```

Внутри структура такая:

```json
{
  "updated": "2026-06-09",
  "groups": [
    {
      "title": "Командование",
      "members": [
        {
          "name": "ФИО / ник",
          "rank": "звание",
          "position": "должность",
          "callsign": "позывной",
          "status": "active",
          "note": "примечание"
        }
      ]
    }
  ]
}
```

Статусы:

- `active` — активен;
- `trainee` — стажёр;
- `reserve` — резерв;
- `leave` — отпуск.

### Вариант 2. Через `admin.html`

1. Открой страницу `admin.html` на сайте.
2. Добавь, измени или удали сотрудников.
3. Нажми `Скачать`.
4. Получишь файл `roster.json`.
5. Замени им файл `data/roster.json` в репозитории.
6. Сделай commit.

Важно: `admin.html` — не настоящая серверная админка. Она не может сама менять сайт без доступа к репозиторию, потому что GitHub Pages — статический хостинг. Это специально сделано без токенов в открытом клиентском коде.

## Как дать доступ на редактирование состава другому человеку

Добавь человека в collaborators репозитория. После этого он сможет менять `data/roster.json` через GitHub и коммитить изменения.

Для личного репозитория: `Settings` → `Collaborators` / `Collaborators & teams` → `Add people`.

## Как смотреть локально

Из-за загрузки `data/roster.json` через `fetch` лучше открывать сайт не двойным кликом по HTML, а через локальный сервер:

```bash
python -m http.server 8000
```

Потом открыть:

```text
http://localhost:8000
```
